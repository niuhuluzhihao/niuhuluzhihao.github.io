# 大模型框架/平台对比与适用场景

本文对比了以下大模型相关框架和平台，并给出了各自适合的应用场景：

- Ollama
- vLLM
- SGLang
- FastChat
- LLaMA Factory
- One-API
- LocalAI

---

## 1. 框架概览

| 框架/平台               | 类型                 | 特点                                               | 适合场景                           | 优势                                      | 局限                                 |
| ----------------------- | -------------------- | -------------------------------------------------- | ---------------------------------- | ----------------------------------------- | ------------------------------------ |
| **Ollama**        | 本地 & 云端 LLM 管理 | 专注 Mac/Windows 本地运行大模型，提供简单 API 接口 | 个人开发者、原型验证、快速本地测试 | 轻量、易用，开箱即用                      | 模型生态相对有限，对 GPU 扩展有限    |
| **vLLM**          | 高性能推理库         | 针对大模型的高吞吐量推理优化                       | 多 GPU 分布式推理、高并发服务      | 高性能，低延迟，支持 tensor parallel      | 仅推理，不管理模型生命周期           |
| **SGLang**        | 多模型统一接口框架   | 提供统一接口对接不同模型                           | 企业级应用、嵌入式服务             | 多模型兼容性强，便于迁移                  | 社区较小，文档资源有限               |
| **FastChat**      | LLM 聊天系统框架     | 提供聊天机器人系统、消息管理、模型调度             | 构建本地或云端聊天系统             | 成熟的聊天系统实现，支持多模型切换        | 主要专注聊天，不通用推理或 embedding |
| **LLaMA Factory** | 模型管理与推理平台   | 对 LLaMA 系列模型做封装和管理                      | 模型训练/推理快速部署              | 易于管理 LLaMA 模型，提供统一接口         | 仅针对 LLaMA 系列                    |
| **One-API**       | 模型接口统一平台     | 类似 OpenAI API 的统一调用接口                     | 企业多模型服务，SaaS 应用          | 统一接口调用不同模型，支持 embedding/chat | 依赖 One-API 平台，可能有限制        |
| **LocalAI**       | 本地部署 LLM 服务器  | 本地部署 OpenAI 兼容接口，支持多种模型             | 本地推理、离线环境、隐私敏感场景   | 支持 OpenAI API 协议，方便替换            | 模型管理需自己处理，部署成本高       |

---

## 2. 深入对比概括

### 2.1 推理性能

- **vLLM** → 专注高性能推理，低延迟，高吞吐量，支持分布式。
- **FastChat / Ollama / LocalAI** → 中等性能，适合单机或小规模并发。
- **SGLang / One-API / LLaMA Factory** → 侧重接口和管理，不是推理优化工具。

### 2.2 模型管理

- **Ollama / LLaMA Factory** → 本地模型管理方便。
- **One-API / SGLang** → 多模型统一管理接口，适合企业。
- **vLLM** → 只关注推理，不管理模型。

### 2.3 部署场景

| 场景               | 推荐框架/平台   |
| ------------------ | --------------- |
| 个人本地测试       | Ollama, LocalAI |
| 企业多模型统一调用 | SGLang, One-API |
| 高并发聊天机器人   | FastChat + vLLM |
| LLaMA 系列专用部署 | LLaMA Factory   |
| 离线部署，隐私敏感 | LocalAI         |

### 2.4 功能对比（API / Chat / Embedding）

| 框架/平台     | Chat | Embedding | Training   | 推理优化           |
| ------------- | ---- | --------- | ---------- | ------------------ |
| Ollama        | ✅   | ❌        | ❌         | ❌                 |
| vLLM          | ❌   | ❌        | ❌         | ✅                 |
| SGLang        | ✅   | ✅        | ❌         | ❌                 |
| FastChat      | ✅   | ❌        | ❌         | ⚠️（多模型调度） |
| LLaMA Factory | ⚠️ | ⚠️      | ✅（微调） | ⚠️               |
| One-API       | ✅   | ✅        | ❌         | ❌                 |
| LocalAI       | ✅   | ✅        | ❌         | ⚠️               |

> ⚠️ 表示功能部分支持或需自定义。

---

## 3. 总结

1. **本地快速原型** → Ollama, LocalAI
2. **高并发推理** → vLLM
3. **聊天系统** → FastChat + vLLM
4. **企业级统一接口** → SGLang, One-API
5. **LLaMA 系列优化** → LLaMA Factory

## vllm调用时候模型资源

### 资源分析

| 组成部分                             | 占用说明                                         | 是否受 `gpu_memory_utilization` 控制 |
| ------------------------------------ | ------------------------------------------------ | -------------------------------------- |
| 1. 模型权重（Model Weights）         | 模型参数（float16/bfloat16），常驻显存           | ❌ 不受控制（必须加载）                |
| 2. KV Cache（PagedAttention 缓存）   | 存储每个 token 的 key/value，随 batch 和长度增长 | ✅ 受 `gpu_memory_utilization` 控制  |
| 3. 临时工作空间（Temporary Buffers） | Attention 计算、softmax、采样等中间结果          | ❌ 不受控制，动态分配                  |
| 4. 显存碎片 & 对齐开销               | CUDA 分配器需要对齐块，可能产生碎片              | ❌ 无法精确预测                        |
| 5. 系统/驱动/其他进程                | NVIDIA 驱动、桌面 GUI、监控工具等                | ❌ 不可控                              |

### 实例说明

举个实际例子（以24G GPU为例）
假设：
    模型：Qwen-1.7B  约3.4GB
    设置： gpu_memory_utilization = 0.8
    总显存：24G

```
总显存：24 GB
│
├── 模型权重：3.4 GB（固定）
├── KV Cache 最多：24 × 0.8 = 19.2 GB → 实际可用 ≈ 19.2 - 3.4 = 15.8 GB？
└── 剩余 20%（4.8 GB）：留给临时计算 + 安全边际
```

### vllm参数解析

- batch_size

  - '喂饭的速度’
- `max_num_seqs`
  - vllm内部的消化能力
  
- `gpu_memory_utilization`

  - 值越大 → 能缓存更多请求的中间状态 → 支持更长上下文或更高并发
  - 值太小 → 浪费显存，降低吞吐
  - 值太大 → 可能 OOM（Out-Of-Memory）

| 参数                              | 含义                           | 控制内容                                                | 作用对象     | 是否包含 Prompt    | 影响资源                    | 典型值           |
| --------------------------------- | ------------------------------ | ------------------------------------------------------- | ------------ | ------------------ | --------------------------- | ---------------- |
| `max_seq_len（其他框架，非LLM)` | 最大序列长度（上下文窗口上限） | 单条请求中**prompt + 生成内容** 的总 token 数上限 | 单条请求     | ✅ 是              | 显存（KV Cache 随长度增长） | 2048, 4096, 8192 |
| `max_tokens`                    | 最大生成 token 数              | 单条请求中**最多生成的新 token 数**               | 单条请求     | ❌ 否（仅输出）    | 显存 + 推理时长             | 128, 256, 512    |
| `max_num_seqs(VLLM中)`          | 最大并发序列数                 | 一个 batch 中**最多同时处理的请求数量**           | 整个推理批次 | —（每条独立计算） | 总显存 + 吞吐量             | 16, 32, 64, 256  |



| 参数                       | 含义                                       | 作用阶段                  | 典型值                          | 备注                                                       |
| -------------------------- | ------------------------------------------ | ------------------------- | ------------------------------- | ---------------------------------------------------------- |
| `max_model_len`          | 模型支持的最大上下文长度（input + output） | 限制单个请求的最大长度    | `4096`,`32768`,`"auto"`   |                                                            |
| `max_num_batched_tokens` | 所有并发请求的总 token 数上限              | 控制 GPU 显存使用，防 OOM | `32768`,`262144`,`524288` | `max_num_batched_tokens ≈ max_num_seqs × max_model_len`` |

```python
import sys
import time
sys.path.append("")

from cal_acc_lora_llm import evaluate_predictions,evaluate_predictions_relaxed_v2
from vllm import LLM, SamplingParams
from transformers import AutoTokenizer
import pandas as pd
import torch
import time
from tqdm import tqdm
import re
import os
from loguru import logger
import json



key_map = {
    "态度": "attitude",
    "骚扰-不再致电": "no_call",
    "骚扰-外呼频次": "freq_call",
    "投诉倾向": "complaint"
}

ID_TO_LABEL = {
    "0": "其他",
    "4": "骚扰-不再致电",
    "5": "态度",
    "6": "骚扰-外呼频次",
    "7": "投诉倾向",
    "8": "投诉倾向&骚扰-不再致电",
    "9": "投诉倾向&骚扰-外呼频次",
    "10": "态度&投诉倾向",
    "11": "态度&骚扰-外呼频次",
    "12": "态度&骚扰-不再致电",
    "13": "骚扰-不再致电&骚扰-外呼频次",
    "14": "态度&骚扰-不再致电&骚扰-外呼频次",
    "15": "态度&投诉倾向&骚扰-不再致电",
    "16": "态度&投诉倾向&骚扰-外呼频次",
    "17": "投诉倾向&骚扰-不再致电&骚扰-外呼频次"
}

def normalize_label(label: str) -> str:
    return "&".join(sorted(label.split("&")))

# key_map的英文 -> 中文的映射
eng2zh = {v: k for k, v in key_map.items()}

# ID_TO_LABEL 的英文 --> label -> id 反查
LABEL_TO_ID = {
    normalize_label(v): k
    for k, v in ID_TO_LABEL.items()
}


def pred_emotion_to_label(pred_emotion):
    """
    输入:
      - dict
      - 或 json 字符串
    输出:
      - (label_id, label_name)
    """
    if pd.isna(pred_emotion):
        return pd.Series(["0", "其他"])

    # 如果是字符串，先反序列化
    if isinstance(pred_emotion, str):
        try:
            pred_emotion = json.loads(pred_emotion)
        except (json.JSONDecodeError, TypeError) as e:
            # 解析失败：视为无命中，返回“其他”
            # 可选：记录日志或打印错误（调试用）
            print(f"JSON 解析失败: {e} | 原始输入: {repr(pred_emotion)}")
            pred_emotion = {"attitude": 0, "no_call": 0, "freq_call": 0, "complaint": 0}

    # 命中的中文标签
    hit_labels = [eng2zh[k] for k, v in pred_emotion.items() if v == 1]

    if not hit_labels:
        label_name = "其他"
    else:
        # 排序保证和 ID_TO_LABEL 完全一致
        label_name = normalize_label("&".join(hit_labels))

    pred_label = LABEL_TO_ID.get(label_name, "0")
    return pd.Series([pred_label, label_name])


def run_inference(
    model_path: str,
    input_excel_path: str,
    output_excel_path: str,
    system_prompt: str,
    batch_size: int = 32,
    dtype: str = "bfloat16",
    gpu_memory_utilization: float = 0.87,
    max_num_seqs: int = 32,
    temperature: float = 0.5,
    top_p: float = 0.9,
    repetition_penalty: float = 1.1,
    max_tokens: int = 2048,
    is_label: bool = True,
    sheet_name:str = ''
):
    """
    使用 vLLM 对 Excel 中的文本进行批量情绪分类推理。

    Args:
        model_path: 模型路径（合并 LoRA 后的 checkpoint）
        input_excel_path: 输入 Excel 文件路径（需包含 'text_a' 列）
        output_excel_path: 输出 Excel 文件路径（会新增 'pred_emotion' 列）
        system_prompt: 系统提示词（情绪分类规则）
        batch_size: 推理批大小
        dtype: 模型精度（如 'bfloat16', 'float16'）
        gpu_memory_utilization: GPU 显存利用率
        max_num_seqs: vLLM 最大并发序列数
        temperature / top_p / repetition_penalty / max_tokens: 采样参数
    """
    logger.info("🔧 正在加载 tokenizer 和模型...")
    tokenizer = AutoTokenizer.from_pretrained(model_path, trust_remote_code=True)
    llm = LLM(
        model=model_path,
        dtype=dtype,
        gpu_memory_utilization=gpu_memory_utilization,
        trust_remote_code=True,
        max_num_seqs=max_num_seqs,
    )

    logger.info("📥 正在读取输入数据...")
    if not sheet_name :
        sheet_name = 0
    df = pd.read_excel(input_excel_path, sheet_name=sheet_name)
    # df = df.head()
    logger.info("📝 正在构建 prompts...")
    prompts = []
    for _, row in df.iterrows():
        user_query = str(row["text_a"]) if pd.notna(row["text_a"]) else ""
        messages = [
            {"role": "system", "content": system_prompt.strip()},
            {"role": "user", "content": user_query},
        ]
        prompt = tokenizer.apply_chat_template(
            messages, tokenize=False, add_generation_prompt=True
        )
        prompts.append(prompt)

    sampling_params = SamplingParams(
        temperature=temperature,
        top_p=top_p,
        repetition_penalty=repetition_penalty,
        max_tokens=max_tokens,
    )

    logger.info(f"🚀 开始批量推理（batch_size={batch_size}）...")
    torch.cuda.synchronize() if torch.cuda.is_available() else None
    start = time.time()

    results = []
    for i in tqdm(range(0, len(prompts), batch_size)):
        batch_prompts = prompts[i : i + batch_size]
        outputs = llm.generate(batch_prompts, sampling_params=sampling_params)
        for out in outputs:
            text = out.outputs[0].text.strip()
            # 移除 <think>...</think> 标签（如果模型输出中有）
            cleaned_text = re.sub(r"<think>.*?</think>", "", text, flags=re.DOTALL)
            results.append(cleaned_text.strip())

    torch.cuda.synchronize() if torch.cuda.is_available() else None
    end = time.time()
    if is_label:
        df["pred_label"] = results
    else:
        df["pred_emotion"] = results

    # 确保输出目录存在
    os.makedirs(os.path.dirname(output_excel_path), exist_ok=True)
    df.to_excel(output_excel_path, index=False)

    logger.info(f"\n✅ 全部生成完成，共 {len(results)} 条。")
    logger.info(
        f"⚙️ 总推理耗时: {end - start:.2f} 秒，平均每条 {(end - start) / len(results):.2f} 秒。"
    )




if __name__ == "__main__":
    # ==============================
    # 路径与配置（集中管理）
    # ==============================
    from prompt import *


     # TODO 回头对于某一个模型，直接给一个列表
    BASE_PATH = '/data/oceanus_share/mzh/inspection_dianxiao_inference/test/resource/benchmark_data/'
    MODEL_BASE_PATH = "/data/oceanus_share/mzh/LLaMA-Factory/saves/Qwen3-1.7B-Base/merge_lora"
    PRE_PATH = "/data/oceanus_share/mzh/inspection_dianxiao_inference/test/resource/benchmark_data_pre/"
    test_date = ''

    # --------------------------需要配置 ----------------------------
    sheet_name = "standard_test_set2"                                                      # 测试集版本
    version = "stress_test"                                                         # 数据版本
    model_path ='train_2026-01-08-23-04-01/checkpoint-840'                    # 模型配置
    lora_rank = 16
    SYSTEM_PROMPT = QIFU_NOTHINK_V12                                           #提示词配置

    # -------------------------------------------------------------------

    model_display_path = model_path.replace("/checkpoint","-checkpoint")
    INPUT_EXCEL =os.path.join(BASE_PATH,test_date,f"{sheet_name}.xlsx")
    MODEL_PATH = os.path.join(MODEL_BASE_PATH,model_path)
    OUTPUT_EXCEL = os.path.join(PRE_PATH,f"{version}/{sheet_name}_{model_display_path}_{lora_rank}.xlsx")
    END_OUTPUT_EXCEL = OUTPUT_EXCEL.replace(".xlsx","_end.xlsx")

    # # 调用推理函数
    run_inference(
        model_path=MODEL_PATH,
        input_excel_path=INPUT_EXCEL,
        output_excel_path=OUTPUT_EXCEL,
        system_prompt=SYSTEM_PROMPT,
        batch_size=400,
        dtype="float16",
        gpu_memory_utilization=0.7,              # 只控制kv cache 的缓存

        temperature=0.01,
        top_p=1.0,
        repetition_penalty=1.0,
        max_tokens=2048,

    )

    data = pd.read_excel(OUTPUT_EXCEL)
    logger.info(f"保存到{OUTPUT_EXCEL}")
    if "attitude" in  data["pred_emotion"][0]:
        data[["pred_label", "label_name"]]=data["pred_emotion"].apply(pred_emotion_to_label)
        data = data.dropna(subset=['true_label'])

    MUL_METRICS_OUTPUT_FILE = MUL_METRICS_OUTPUT_FILE.replace(f"{version}/",f"{version}/mul_metric_")
    print(f"保存到 {MUL_METRICS_OUTPUT_FILE}")
    evaluate_predictions(
        input_excel_path=END_OUTPUT_EXCEL,
        output_excel_with_pred_path=OUTPUT_WITH_LABEL_FILE,

    )

    evaluate_predictions_relaxed_v2(
        input_excel_path=END_OUTPUT_EXCEL,

        is_label=True,  # 注意：如果 pred_emotion 是文本，应设为 False
```
