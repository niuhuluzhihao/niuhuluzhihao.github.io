# 提交代码库命令

```bash
git pull -r                          # 将工作区更新至当前最新版本
git add .                            # 将文件从工作区添加至暂存区
git commit -m ""                     # 将暂存区代码提交至本地仓库
git push origin HEAD:refs/for/master # 将代码提交至远程仓库
```

# 其他指令

## 创建仓库

```bash
git init   # 创建一个空的 git 仓库或重新初始化现有仓库
git clone  # 拷贝远程仓库，即下载一个项目
```

## 配置用户名

```bash
git config --global user.name "your_name"           # 全局配置，当前项目只需要去掉global
git config --global user.email "your_email@example.com"  # 全局配置，邮箱必须和github账号绑定
git config --global --list                           # 查看全局配置
git config --global --unset user.email               # 删除邮箱地址
git config --global --unset user.name                # 删除用户名称
git config --list                                    # 查看所有配置
```

## 提交与修改

```bash
git add                    # 添加到暂存区
git status                 # 查看仓库当前状态
git diff                   # 比较暂存区和工作区的差异，即查看修改的内容
git commit                 # 提交暂存区到仓库
git commit --amend         # 追加代码并添加追交的提交说明
```

### git reset - 回退版本

```bash
git reset HEAD^                    # 回退所有内容到上一个版本
git reset --hard HEAD~5            # 回退到之前5个版本（重要）
git reset HEAD^5                   # 回退到之前5个版本（会返回最新）
git reset HEAD^ hello.php          # 回退hello.php文件的版本到上一个版本
git reset 052e                     # 回退到指定的版本
git reset --soft                   # 回退到某个版本，只回退commit信息，不恢复到index
git reset --hard                   # 彻底回退到某个版本，本地源码也会变为上一个版本
```

```bash
git mv                    # 移动和重命名工作区文件
git rm                    # 将文件从暂存区和工作区中删除
git stash                 # 将所有未提交的修改保存至堆栈中
git stash apply           # 恢复，但stash内容不删除，需要手动git stash drop
git stash pop             # 恢复的同时把stash内容也删除
git log                   # 查看自己提交的信息
```

## 提交日志

```bash
git log                   # 查看历史提交记录
```

## 远程操作

```bash
git remote                # 远程仓库操作
git fetch                 # 从远程获取代码库
git pull                  # 下载远程代码并合并
git pull origin master    # 远程分支与当前分支合并
git pull origin master:brantest  # 远程分支与本地brantest分支合并
git push                  # 上传远程代码并合并
```

## 分支管理

```bash
git branch branchname     # 创建分支
git checkout branchname   # 切换分支
git merge                 # 合并分支
git branch -d branchname  # 删除分支（删除前需要先切换到其他分支）
git cherry-pick <commit id>  # 选择某一个分支中的一个或几个commit来操作
git format-patch          # 对某次提交生成patch（补丁）
git rebase                # 变基操作
```

### pull与rebase的区别

```bash
git pull = git fetch + git merge
git pull --rebase = git fetch + git rebase
```

# 同时配置多个GitHub和GitLab账号（Windows需要在Git Bash中运行）

## 配置用户名

```bash
git config -- user.name "your_name"           # 不要配置global
git config --user.email "your_email@example.com"  # 不要配置global
```

## 执行如下命令

```bash
ssh-keygen -t ed25519 -C "18862776370@163.com" -f ~/.ssh/id_ed25519_github
ssh-keygen -t ed25519 -C "your_gitlab_email@example.com" -f ~/.ssh/id_ed25519_gitlab
```

## 查看生成的密钥

```bash
ls -la ~/.ssh/
```

## 复制到对应的GitHub和GitLab账号中

- `id_ed25519_github`
- `id_ed25519_github.pub`

## 验证是否配置成功

1. 启动ssh agent，在终端运行

```bash
eval "$(ssh-agent -s)"
```

2. 将私钥添加到Agent

```bash
ssh-add ~/.ssh/id_ed25519_github
ssh-add ~/.ssh/id_ed25519_gitlab
```

3. 测试链接

```bash
ssh -T git@github.com
```
